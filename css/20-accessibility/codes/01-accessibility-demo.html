CSS 无障碍访问示例

1. 焦点样式（Focus Styles）

为键盘用户提供清晰的焦点指示
焦点样式示例： button:focus {
  outline: 3px solid #3498db;
  outline-offset: 2px;
}

/* 仅在键盘导航时显示 */
button:focus-visible {
  outline: 3px solid #2ecc71;
  outline-offset: 2px;
}

使用 Tab 键导航，查看焦点样式：

💡 提示：永远不要移除 outline，除非提供了替代的焦点样式。

2. 颜色对比度

确保文本与背景之间有足够的对比度
✅ 好的对比度（7.5:1）：
这段文字有足够的对比度，易于阅读。
❌ 差的对比度（1.5:1）：
这段文字对比度不足，难以阅读。
💡 WCAG 标准：
- AA 级别：文本至少 4.5:1，大文本 3:1
- AAA 级别：文本至少 7:1，大文本 4.5:1

3. 屏幕阅读器专用内容

为屏幕阅读器提供额外的上下文信息
视觉隐藏但屏幕阅读器可读： .sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border-width: 0;
}
💡 提示：使用 aria-hidden="true" 隐藏装饰性内容，避免屏幕阅读器重复读取。

4. 减少动画（prefers-reduced-motion）

尊重用户的动画偏好设置
尊重用户偏好： @media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
💡 提示：在系统设置中开启"减少动态效果"，动画会自动禁用。

5. 高对比度模式

支持高对比度模式
高对比度媒体查询： @media (prefers-contrast: high) {
  .button {
    border: 2px solid currentColor;
  }
}

这个容器在高对比度模式下会有不同的样式。

在系统设置中开启高对比度模式查看效果。

6. 表单无障碍

确保表单对所有用户可用
用户名至少需要 3 个字符
请输入有效的邮箱地址
💡 提示：使用 aria-describedby 关联提示信息，aria-invalid 标记错误状态。

7. 链接和按钮

确保交互元素易于识别和操作
❌ 避免的做法：
点击这里 （链接没有下划线，难以识别）
✅ 推荐的做法：
查看更多信息 （链接有下划线，清晰可见）
💡 提示：链接应该有清晰的下划线或背景色，确保用户能够识别。

8. 禁用状态

清晰地表示禁用状态
禁用状态样式： button:disabled {
  background: #bdc3c7;
  cursor: not-allowed;
  opacity: 0.6;
}
💡 提示：使用 aria-disabled="true" 提供语义化的禁用状态。

9. 响应式字体

使用相对单位确保文本可缩放
推荐使用相对单位： body {
  font-size: 100%; /* 或 16px */
}

h1 {
  font-size: 2rem; /* 相对于根元素 */
}

.text {
  font-size: clamp(1rem, 2vw, 1.5rem); /* 响应式 */
}
💡 提示：避免使用固定像素值，使用 rem、em 或 vw 等相对单位。

10. 无障碍检查清单

CSS 无障碍要点总结
  • ✅ 提供清晰的焦点样式
  • ✅ 确保足够的颜色对比度
  • ✅ 支持键盘导航
  • ✅ 尊重用户的动画偏好
  • ✅ 使用相对单位设置字体大小
  • ✅ 不要仅依赖颜色传达信息
  • ✅ 为屏幕阅读器提供隐藏内容
  • ✅ 确保交互元素有足够大的点击区域
