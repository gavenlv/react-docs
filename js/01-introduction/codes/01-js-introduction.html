JavaScript 三种引入方式演示

1. 行内脚本（Inline Script）

直接在 HTML 标签的事件属性中写 JavaScript 代码
HTML： <button onclick="alert('你点击了按钮！')">点击我</button>
注意：行内脚本不推荐在实际项目中使用，仅用于快速测试

2. 内部脚本（Internal Script）

在 HTML 文件的 <script> 标签中写 JavaScript
HTML： <script>
  function sayHello() {
    alert('你好，世界！');
  }
</script>

<button onclick="sayHello()">点击问好</button>

3. 外部脚本（External Script）⭐ 推荐

将 JavaScript 写在单独的 .js 文件中，然后在 HTML 中引入
index.html： <script src="js/main.js"></script>

js/main.js： const btn = document.getElementById('myBtn');
btn.addEventListener('click', () => {
  alert('外部脚本被执行！');
});

对比总结

方式 优点 缺点 适用场景
行内脚本 简单直接 难以维护、无法复用 临时测试
内部脚本 集中管理、无需额外文件 无法跨页面复用 单页面应用
外部脚本 可复用、易维护、可缓存 需要额外请求 所有项目（推荐）
