## 📖 第一部分：数制基础

### 1.1 什么是数制

**数制**（Number System）是用一组固定的数字符号和统一的规则来表示数值的方法。

#### 数制的要素

| 要素 | 说明 |
|------|------|
| **数码** | 数制中使用的基本符号 |
| **基数** | 数制中数码的个数 |
| **位权** | 每个位置所代表的值 |

### 1.2 常见数制

| 数制 | 基数 | 数码 | 表示方法 | 举例 |
|------|------|------|----------|------|
| **二进制** | 2 | 0, 1 | B或下标2 | 1010B 或 (1010)₂ |
| **八进制** | 8 | 0-7 | O或下标8 | 12O 或 (12)₈ |
| **十进制** | 10 | 0-9 | D或下标10 | 10D 或 (10)₁₀ |
| **十六进制** | 16 | 0-9, A-F | H或下标16 | 1AH 或 (1A)₁₆ |

### 1.3 为什么计算机使用二进制

计算机内部采用二进制的原因：

| 原因 | 说明 |
|------|------|
| **易于实现** | 电子元件的两种状态（开关、高低电平）正好对应0和1 |
| **运算简单** | 二进制运算规则简单，便于硬件实现 |
| **可靠性高** | 只有两种状态，抗干扰能力强 |
| **逻辑运算方便** | 与逻辑代数的真/假对应 |

## 🔢 第二部分：数制转换

### 2.1 十进制转二进制

#### 方法：除2取余，逆序排列

**步骤**：
1. 将十进制数除以2，记录余数
2. 将商继续除以2，记录余数
3. 重复步骤2，直到商为0
4. 将所有余数从下往上排列

**示例**：将十进制25转换为二进制

```
25 ÷ 2 = 12 ... 余数 1
12 ÷ 2 = 6  ... 余数 0
6 ÷ 2 = 3   ... 余数 0
3 ÷ 2 = 1   ... 余数 1
1 ÷ 2 = 0   ... 余数 1

从下往上读：11001
所以 25₁₀ = 11001₂
```

**验证**：1×2⁴ + 1×2³ + 0×2² + 0×2¹ + 1×2⁰ = 16 + 8 + 0 + 0 + 1 = 25 ✓

#### 练习

| 十进制 | 计算过程 | 二进制结果 |
|--------|----------|------------|
| 10 | 10÷2=5...0, 5÷2=2...1, 2÷2=1...0, 1÷2=0...1 | 1010 |
| 15 | 15÷2=7...1, 7÷2=3...1, 3÷2=1...1, 1÷2=0...1 | 1111 |
| 20 | 20÷2=10...0, 10÷2=5...0, 5÷2=2...1, 2÷2=1...0, 1÷2=0...1 | 10100 |

### 2.2 二进制转十进制

#### 方法：按权展开求和

**公式**：将每一位乘以对应的位权，然后求和

**位权表**：

| 位置 | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|------|---|---|---|---|---|---|---|---|
| 位权 | 2⁷ | 2⁶ | 2⁵ | 2⁴ | 2³ | 2² | 2¹ | 2⁰ |
| 值 | 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 |

**示例**：将二进制11001转换为十进制

```
位置：  4  3  2  1  0
数值：  1  1  0  0  1

计算：1×2⁴ + 1×2³ + 0×2² + 0×2¹ + 1×2⁰
    = 16 + 8 + 0 + 0 + 1
    = 25

所以 11001₂ = 25₁₀
```

**常用2的幂次**：

| 幂次 | 值 | 幂次 | 值 |
|------|-----|------|-----|
| 2⁰ | 1 | 2⁶ | 64 |
| 2¹ | 2 | 2⁷ | 128 |
| 2² | 4 | 2⁸ | 256 |
| 2³ | 8 | 2⁹ | 512 |
| 2⁴ | 16 | 2¹⁰ | 1024 |
| 2⁵ | 32 | | |

### 2.3 二进制与八进制转换

#### 二进制转八进制

**方法**：从右向左，每3位一组，不足补0，分别转换为八进制

**示例**：将二进制1011010转换为八进制

```
二进制：  001  011  010    （从右向左分组，不足补0）
          ↓    ↓    ↓
八进制：  1    3    2

所以 1011010₂ = 132₈
```

**对照表**：

| 二进制 | 八进制 |
|--------|--------|
| 000 | 0 |
| 001 | 1 |
| 010 | 2 |
| 011 | 3 |
| 100 | 4 |
| 101 | 5 |
| 110 | 6 |
| 111 | 7 |

#### 八进制转二进制

**方法**：每一位八进制数转换为3位二进制数

**示例**：将八进制132转换为二进制

```
八进制：  1    3    2
          ↓    ↓    ↓
二进制：  001  011  010

所以 132₈ = 1011010₂
```

### 2.4 二进制与十六进制转换

#### 二进制转十六进制

**方法**：从右向左，每4位一组，不足补0，分别转换为十六进制

**示例**：将二进制11010110转换为十六进制

```
二进制：  1101  0110    （从右向左分组，不足补0）
           ↓     ↓
十六进制： D     6

所以 11010110₂ = D6H
```

**对照表**：

| 二进制 | 十六进制 | 二进制 | 十六进制 |
|--------|----------|--------|----------|
| 0000 | 0 | 1000 | 8 |
| 0001 | 1 | 1001 | 9 |
| 0010 | 2 | 1010 | A |
| 0011 | 3 | 1011 | B |
| 0100 | 4 | 1100 | C |
| 0101 | 5 | 1101 | D |
| 0110 | 6 | 1110 | E |
| 0111 | 7 | 1111 | F |

#### 十六进制转二进制

**方法**：每一位十六进制数转换为4位二进制数

**示例**：将十六进制D6转换为二进制

```
十六进制： D     6
           ↓     ↓
二进制：  1101  0110

所以 D6H = 11010110₂
```

### 2.5 十进制转八进制/十六进制

#### 方法一：间接转换

先转换为二进制，再转换为八进制或十六进制

#### 方法二：直接转换

**十进制转八进制**：除8取余，逆序排列

**示例**：将十进制100转换为八进制

```
100 ÷ 8 = 12 ... 余数 4
12 ÷ 8 = 1  ... 余数 4
1 ÷ 8 = 0   ... 余数 1

从下往上读：144
所以 100₁₀ = 144₈
```

**十进制转十六进制**：除16取余，逆序排列

**示例**：将十进制255转换为十六进制

```
255 ÷ 16 = 15 ... 余数 15 (F)
15 ÷ 16 = 0   ... 余数 15 (F)

从下往上读：FF
所以 255₁₀ = FFH
```

## 📝 第三部分：字符编码

### 3.1 ASCII编码

**ASCII**（American Standard Code for Information Interchange，美国信息交换标准代码）是最基本的字符编码。

#### ASCII码特点

- 使用7位二进制数表示字符
- 共可表示128个字符（0-127）
- 包括：控制字符、数字、字母、符号

#### 常用ASCII码表

| 字符类型 | 字符范围 | ASCII码范围 |
|----------|----------|-------------|
| 控制字符 | NUL, CR, LF等 | 0-31 |
| 空格 | 空格 | 32 |
| 数字 | 0-9 | 48-57 |
| 大写字母 | A-Z | 65-90 |
| 小写字母 | a-z | 97-122 |

#### 重要ASCII码记忆

| 字符 | ASCII码（十进制） | ASCII码（二进制） |
|------|-------------------|-------------------|
| '0' | 48 | 0110000 |
| 'A' | 65 | 1000001 |
| 'a' | 97 | 1100001 |

#### ASCII码规律

1. **数字字符**：'0'-'9' 的ASCII码是连续的（48-57）
2. **大写字母**：'A'-'Z' 的ASCII码是连续的（65-90）
3. **小写字母**：'a'-'z' 的ASCII码是连续的（97-122）
4. **大小写关系**：小写字母比对应大写字母大32

**应用**：
- 大写字母转小写：ASCII码 + 32
- 小写字母转大写：ASCII码 - 32

### 3.2 汉字编码

#### 汉字编码的发展

| 编码标准 | 特点 | 字符数 |
|----------|------|--------|
| **GB2312** | 1980年发布，基本汉字 | 6763个汉字 |
| **GBK** | 扩展GB2312，包含繁体 | 20000+汉字 |
| **GB18030** | 最新标准，最全面 | 70000+汉字 |
| **Unicode** | 国际统一编码 | 覆盖全球文字 |

#### 汉字编码过程

```
汉字 → 输入码（拼音、五笔）→ 机内码（存储）→ 输出码（显示）
```

| 编码类型 | 说明 | 举例 |
|----------|------|------|
| **输入码** | 键盘输入的编码 | 拼音"zhong" |
| **机内码** | 计算机内部存储的编码 | 2字节二进制 |
| **字形码** | 显示/打印的点阵编码 | 16×16点阵 |

#### 汉字存储空间

- 一个汉字占2个字节（GBK编码）
- 一个英文字符占1个字节（ASCII编码）

### 3.3 Unicode编码

**Unicode**（统一码）是国际组织制定的字符编码标准，旨在统一全球所有文字的编码。

#### Unicode特点

- 为每种语言的每个字符设定统一的编码
- 解决不同编码标准不兼容的问题
- 常用编码方式：UTF-8、UTF-16、UTF-32

#### UTF-8编码

- 可变长编码，1-4个字节
- 兼容ASCII码
- 英文字符占1字节，汉字通常占3字节

## 💾 第四部分：存储单位

### 4.1 基本存储单位

| 单位 | 英文 | 说明 |
|------|------|------|
| **位** | bit (b) | 最小单位，一个二进制位（0或1） |
| **字节** | Byte (B) | 基本单位，1 Byte = 8 bit |

### 4.2 存储单位换算

```
换算关系（1024进制）：

1 Byte (B) = 8 bit (b)
1 KB (千字节) = 1024 B = 2¹⁰ B
1 MB (兆字节) = 1024 KB = 2²⁰ B
1 GB (吉字节) = 1024 MB = 2³⁰ B
1 TB (太字节) = 1024 GB = 2⁴⁰ B
```

### 4.3 常见数据大小

| 数据类型 | 大小（约） |
|----------|------------|
| 一个英文字符 | 1 Byte |
| 一个汉字 | 2 Bytes |
| 一张普通照片 | 2-5 MB |
| 一首MP3歌曲 | 3-5 MB |
| 一部电影 | 1-2 GB |

### 4.4 计算示例

**例1**：一张照片大小为4MB，等于多少KB？

```
4 MB = 4 × 1024 KB = 4096 KB
```

**例2**：一个U盘容量为8GB，可以存储多少张2MB的照片？

```
8 GB = 8 × 1024 MB = 8192 MB
照片数量 = 8192 ÷ 2 = 4096 张
```

**例3**：一篇文章有5000个汉字，需要多少存储空间？

```
一个汉字占2字节
5000个汉字 = 5000 × 2 = 10000 字节 = 10000 ÷ 1024 ≈ 9.77 KB
```

## ⚠️ 考点总结

### 高频考点

#### 考点一：数制转换

**题型**：选择题、填空题、计算题

**解题技巧**：
- 十进制转二进制：除2取余，逆序排列
- 二进制转十进制：按权展开求和
- 二进制转八进制：3位一组
- 二进制转十六进制：4位一组

**典型例题**：
> 将十进制25转换为二进制
> 答案：11001

#### 考点二：ASCII码

**题型**：选择题

**解题技巧**：
- 记住 '0'=48, 'A'=65, 'a'=97
- 大小写相差32

**典型例题**：
> 字符'A'的ASCII码是65，字符'a'的ASCII码是？
> 答案：65 + 32 = 97

#### 考点三：存储单位换算

**题型**：选择题、计算题

**解题技巧**：
- 记住 1KB = 1024B, 1MB = 1024KB
- 注意是1024进制

**典型例题**：
> 2MB等于多少KB？
> 答案：2 × 1024 = 2048 KB

#### 考点四：汉字存储

**题型**：选择题、计算题

**解题技巧**：
- 一个汉字占2字节
- 一个英文字符占1字节

**典型例题**：
> 存储1000个汉字需要多少字节？
> 答案：1000 × 2 = 2000 字节
